在今天前，我會用到的 Vibe Coding 大多僅限於正常和 Claude 進行對話後生成程式片段，經歷過以前純手刻網頁的時代，這樣做對我而言已經是一種如飛似般的效率提升。

但其實做前端最困擾我的一直都不是技術(前端能有什麼技術哈)，而是介面，我真的是一點藝術腦袋都沒有，連想一個個人網頁的首頁都可以想兩三天。

雖然如此，但我還是對自己的 UI/UX 稍稍有點自豪的，這就不得不提到我的朋友 Yeci 老哥，他以前寫的個人網頁那叫一個賞心悅目，本就不整齊的排版加上那隨時都會飛走的按鈕，屬於是屬於是了。

### 安裝 Plugins

在一番詢問之後，他說他用了一些很屌的 Claude Code Plugins，分別是 `superpowers@claude-plugins-official`, `frontend-design@claude-plugins-official`, `brainstorm@MadeByTokens-marketplace`

聽了之後我立刻就準備下載，結果打開 Plugins 列表一看，啥都沒有 (我用的是 VS Code 的 `Claude Code for VS Code` 擴充)

![image](https://hackmd.io/_uploads/BJYtcPT3Zl.png)

周旋了好一陣子，原來 Plugin 商店還要先註冊 wtf

![image](https://hackmd.io/_uploads/HksljwTnWe.png)
![image](https://hackmd.io/_uploads/S1PWovan-l.png)

註冊之後就有了，以下是我用的商店 Repo

```
https://github.com/MadeByTokens/claude-code-plugins-madebytokens.git
https://github.com/anthropics/claude-plugins-official.git
```

註冊跟安裝的話根據用的平台有很多不同指令，這個就自己去對應 Plugin 的 Repo 找說明書了。

### 下 Prompt

裝好之後終於可以開工了，把長長的需求寫完之後他就開始工作了，這時候特別的來了，他開始詢問我一些細節，比如我網頁想怎麼呈現，眾所周知我的網頁放了一個沒啥用處的假終端機，所以 Claude 提了三個方向:

```
A) 讓終端機成為網頁唯一的互動方式
B) 用戶可以點擊畫面，但只會產生指令至終端機，仍需要和終端機互動
C) 中庸，兩邊都要
```

很顯然的我選了 C，不過像這樣對話去推進我是第一次體驗，可還沒等我驚嘆完， Claude 又突然打開了一個 localhost，讓我進去選設計。

我草，我沒看錯嗎? 選設計

我一直認為 AI 設計只能先生再改，這個 Plugin 直接讓 Claude 現場做了一個 demo，我還可以先看了之後再選要哪個，媽耶這會不會太爽。

![image](https://hackmd.io/_uploads/ryq9Tw63-l.png)
(我沒有截到圖，用 Yeci 的示意)

其實後面調整還花了很多時間，要一次生到我的胃口上沒有這麼簡單，但這是我第一次把整個專案設計全部丟給 AI，以前都只會小部分請 AI 改，這次使用下來的感覺，AI 搭配 Plugins/Skills 真的變得很強，把 Prompt 變成一個可供分享的文件形式，就像讓 AI 去閱讀說明書，可以更精準的執行對應操作，不用全靠它猜，非常看好這個發展方向。
